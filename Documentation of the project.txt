AIM:

To design and implement a console-based Railway Booking System in Python that applies Object-Oriented Programming principles to simulate a real-world railway reservation workflow, including train allocation, seat booking, ticket generation, fare calculation, and transaction logging.

PROBLEM STATEMENT:
Build a terminal-driven railway booking application where the user can enter passenger details, source station, destination station, journey date, and required number of seats.
The system must automatically allocate a train based on route availability, calculate total fare, generate PNR number, allocate coach and seat numbers, print a formatted railway ticket, and store booking records into text files.
The application must validate inputs, handle invalid entries gracefully using exception handling, and prevent abrupt program termination.

INPUTS:
Terminal menu option (1 / 2 / 3)
Passenger name
Passenger age
ID proof number
Source station
Destination station
Journey date
Number of seats required
Booking confirmation (yes/no)

OUTPUTS:
Display of allotted train details
Fare details including total ticket cost
Railway ticket with:
PNR Number
Passenger details
Train details
Coach number
Seat numbers
Total fare
Booking confirmation message
Error message for invalid input or unavailable seats
Booking details stored in:
bookings.txt
tickets.txt

CONCEPTS USED:
Object-Oriented Programming
Passenger Class
Train Class
Booking Class
Encapsulation
Private seat variable (__seats)
Inheritance
Booking class inherits Train class
Polymorphism
display() method implementation
Exception Handling
Invalid input handling using try-except
File Handling
Writing booking details into text files
Loops and Conditional Statements
Menu-driven while loop and booking logic
Random Module
Random PNR, coach, and seat generation

Step 1:

Start the program.

Load predefined train data including:

Train Number

1 - Book Ticket

3 - Exit

Accept user choice.

Step 6:

If choice = 1:

a.

Accept passenger details:

Name
Age

Source Station

Number of Seats Required
